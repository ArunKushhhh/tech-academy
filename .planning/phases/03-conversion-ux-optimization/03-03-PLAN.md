# Plan 03-03: Static Export Configuration & Build Verification

## Objective
Configure Next.js for static HTML/CSS/JS export (compatible with EmailJS and the existing stack), verify the build succeeds with zero errors, and confirm the `out/` directory is generated.

## Tasks

### Task 1: Update next.config.ts for static export

<read_first>
- next.config.ts (current content — currently just `const nextConfig: NextConfig = {};`)
- sections/educator.tsx (check if it uses next/image — it does: `import Image from "next/image"`)
</read_first>

<action>
Replace the content of `next.config.ts` with:

```ts
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  output: "export",
  images: {
    unoptimized: true,
  },
};

export default nextConfig;
```

**Why `images.unoptimized: true`:** The educator section uses `next/image` (`import Image from "next/image"`). Next.js Image Optimization requires a server — it is incompatible with static export. Setting `unoptimized: true` tells Next.js to skip optimization and serve images as-is, which works perfectly in a static context.
</action>

<acceptance_criteria>
- `next.config.ts` contains `output: "export"`
- `next.config.ts` contains `images:` config block
- `next.config.ts` contains `unoptimized: true`
</acceptance_criteria>

### Task 2: Run pnpm build and verify

<read_first>
- next.config.ts (verify config is saved correctly before running build)
- package.json (confirm build script — should be `next build`)
</read_first>

<action>
Run the build command:
```bash
pnpm build
```

Expected outcome:
- Build exits with code 0 (success)
- `out/` directory is created at project root
- `out/index.html` exists
- No TypeScript errors, no Next.js build errors

If build fails:
1. Read the error output carefully
2. Common issues and fixes:
   - `"use client" required` — add `"use client"` to any component using hooks that's missing it
   - Image optimization error → already handled by `unoptimized: true`  
   - `useSearchParams` without Suspense → not expected in this project
   - Missing env var types → TypeScript will accept `process.env.NEXT_PUBLIC_*` with `!` non-null assertion
3. Fix the error and re-run build
4. Report what was fixed
</action>

<acceptance_criteria>
- `pnpm build` exits with code 0
- `out/` directory exists at project root after build
- `out/index.html` exists
- Build output shows no TypeScript errors
- Build output shows no Next.js static export incompatibility errors
</acceptance_criteria>

## must_haves
- `next.config.ts` has `output: "export"` and `images.unoptimized: true`
- `pnpm build` succeeds (exit code 0)
- `out/index.html` exists after build
